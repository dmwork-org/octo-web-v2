# Security Policy

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues.

Report security issues by email:

```text
security@mininglamp.com
```

You should receive a response within 72 hours. If you do not, please follow up by email.

Please include as much of the following information as possible:

- Type of issue, such as XSS, injection, authentication bypass, or sensitive data exposure.
- Affected branch, commit, or release.
- Affected source files or routes.
- Steps to reproduce.
- Proof of concept, if available.
- Impact and suggested mitigation, if known.

## Disclosure Policy

We follow coordinated disclosure. Please give maintainers a reasonable amount of time to investigate and release a fix before public disclosure.

After a fix is available, we may publish a GitHub security advisory and credit the reporter unless anonymity is requested.

## Supported Versions

| Version | Supported |
| ------- | --------- |
| latest  | Yes       |
| older   | No        |
